# Network Device Configuration Templates

Проект содержит шаблоны конфигураций и инструменты генерации настроек сетевого оборудования различных вендоров

## Цель проекта

- Создать библиотеку шаблонов, которые могут быть использованы для настройки сетевого оборудования

## Основные принципы

- Шаблоны хранятся в текстовом виде (Jinja2 или аналог)  
- В шаблонах рекомендуется использовать **YAML-docstring** в начале файла, заключенный в `{#--- ... ---#}`:
  - Поля шаблона (vars), их типы, обязательность, значения по умолчанию, примеры и описания переменных.
  - Это позволяет инструментам генерировать подсказки, валидировать ввод, строить современный пользовательский интерфейс.
- Генерация конфигураций выполняется из переменных (вручную или при помощи утилиты `tools/render-template-interactive.py`)
  - Скрипт парсит YAML-docstring и обеспечивает интерактивный ввод с проверками типа, выводом описаний и примеров.
  - Шаблоны без YAML-блока также поддерживаются (режим полной обратной совместимости).
- Поддержка как первичного bootstrap, так и последующих донастроек / шаблонов для типовых задач  
- Документация конкретных сценариев и инструкций — в папке `docs/`

## Корректный формат YAML-docstring в шаблонах

```jinja
{#---
name: <уникальное имя шаблона>
description: <описание, когда и зачем использовать>
vars:
  <имя_переменной>:
    type: <str|int|bool|list[str]|cidr|ip|...>
    required: <true|false>
    desc: <описание переменной>
    default: <значение по умолчанию, если есть>
    example: <пример значения>
---#}
```
